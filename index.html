Hiroshima AI Assurance Protocol

Verified before action.
Preserved after.

HAAP is the proposed accountability layer between business rules and transport protocols. It does not replace HTTP, gRPC, A2A, or MCP. It specifies how requirements, evidence, verification outcomes, and adoption states can travel together.

Gate
Before state change

Required conditions and evidence are checked before an AI decision is adopted.

Separation
Claims are not verification

Inspection claims are evaluated against external evidence, not accepted as self-proof.

Record
Append-only assurance

Decision, evidence reference, result, and responsibility state are preserved for review and replay.

Projects

Phase 1 is not a slide deck.
It has a working PoC.

The first domain implementation is the Logistics Profile for pharmaceutical cold-chain handoff. It validated the core verification and chain-stop engine, while leaving production integration and cross-company interoperability for later phases.

Validated in Phase 1: ruleset binding, evidence field inspection, independent metric recalculation, verdict generation, downstream chain-stop control, and digest-based replay verification.

Not yet validated: sensor authenticity, production deployment, cross-company API exchange, multi-implementation interoperability, and full wire-format specifications.
